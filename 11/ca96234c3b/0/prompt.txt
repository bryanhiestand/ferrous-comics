Since your main function isn't async, you should lean toward using a purely synchronus HTTP client, you've pulled in reqwest, which is definitely the best async http client, but you're using the blocking version of it: reqwest::blocking::Client

Internally, the blocking client for reqwest is very heavy handed, it spins up a dedicated async runtime, tokio, and then spawns a blocking thread.

In this circumstance, a non async main, using the ureq library, would be a much better fit, and much more efficient and speedy
