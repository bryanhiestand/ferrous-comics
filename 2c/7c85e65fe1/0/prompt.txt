Trimming XKCD_SMTP_USERNAME/XKCD_SMTP_PASSWORD changes credential semantics (it becomes impossible to use leading/trailing spaces in either value) and can also turn an all-whitespace env var into Some(""), which then passes validation and results in an attempted auth with empty credentials. Consider preserving these values verbatim, or if whitespace should be ignored, treat a post-trim empty string as unset (or surface a clear validation error).
